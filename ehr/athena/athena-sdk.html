SDK / Ambient Notes

Setup: New users and clinics do not need to be pre-configured in iScribe. When a provider launches the Athena One app and starts recording their first job, the clinic and user will be automatically created via SDK integration.

New Clinic Creation Logic

iScribe checks for an existing clinic with the same Athena Practice ID and verifies the Partner field is set to Athena SDK. If both conditions are unmet, a new clinic is created.

New User Creation Logic

iScribe searches within the matched clinic to see if a user with the same EHR Username exists. If no match is found, a new user is created with the following attributes:

Support Process
Modification of Provider Preferences
  • Clarify the request and confirm provider expectations.
  • New or custom prompts cannot be created for SDK users.
  • Provide feedback and educate providers on best dictation practices.
Output Errors
  • Check NPS configuration alignment with workflow.
  • Review confidence score for the Correlation ID.
  • Confirm AI confidence matches expected results. The confidence score must be at least 20%. If no content is saved in the AI Summary, this is due to the overall confidence score. If some sections generated in the AI Summary, those sections would have saved in Athena. The provider should see these.
Jobs Not Received
  • Validate provider workflow and device usage.
  • Review Correlation ID & confidence score.
  • Prepare escalation to Level 3 with provider details, workflow, and findings.
SDK Ticketing Process
  1. Ticket Title: Format as PX SUPPORT-[Ticket Number] and assign to Athena SDK inbox.
  2. Ticket Description Format:
    • Clinic: [Clinic name or Athena Practice ID]
    • User: [Provider name / EHR username]
    • Issue: [Detailed description]
