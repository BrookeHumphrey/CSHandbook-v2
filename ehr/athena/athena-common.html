Common Athena Issues and Fixes

This page outlines frequently encountered problems in Athena workflows and how to resolve them.

1. iScribe Dictations Not Appearing

Ensure that the provider’s iScribe app is synced with Athena. If encounters are missing:

  • Confirm Wi-Fi or cellular connectivity.
  • Check if dictations were saved under the correct user ID.
  • Have the provider refresh the encounter list.

2. Note Does Not Pull Forward

This usually occurs if the previous encounter was closed or finalized. To correct:

  • Open the prior encounter and verify it’s in an editable state.
  • Use “Copy Previous Note” from the documentation toolbar.

3. AI-Generated Text Missing or Incomplete

If Athena AI fails to populate text:

  • Verify AI assistance is enabled for the provider.
  • Confirm that the template contains active AI fields.
  • Check for browser caching — clear cache and retry.

4. Error Messages (e.g., “Enum Value Invalid”)

This typically indicates that the system field expects a specific standardized value. Review dropdown or entry fields to ensure data matches Athena’s valid options.

5. Auth0 Integration & Login Guidance

In the event that a login issue arises, users can switch from Wi-Fi to LTE, allowing them to continue their work. Please do not remove any previously established URLs. The IP addresses mentioned below are new additions to avoid any potential issues.

Why this is necessary: We will be changing our authentication method to better support our users and enhance overall security. As part of these changes, we have identified a few IP addresses and domains that may need to be added to the whitelist to prevent the firewall from blocking them.

  • Whitelist all subdomains for https://iscribehealth.com: *.iscribehealth.com
  • Authentication process: prod-iscribehealth.us.auth0.com
  • Images used in the application: cdn.auth0.com
  • App validation with Auth0 domain: app-site-association.cdn-apple.com
  • Login page logos: cdn.prod.website-files.com
